---
title: 'Message Queue Pattern'
description: 'Use Bull as a persistent message queue for reliable server-to-server communication'
---

Bull can be used for persistent message queues. This is a quite useful feature in some use cases. For example, you can have two servers that need to communicate with each other. By using a queue the servers do not need to be online at the same time, so this creates a very robust communication channel.

You can treat `add` as **send** and `process` as **receive**.

## Implementation

<CodeGroup>

```js Server A
const Queue = require('bull');

const sendQueue = new Queue('Server B');
const receiveQueue = new Queue('Server A');

receiveQueue.process(function (job, done) {
  console.log('Received message', job.data.msg);
  done();
});

sendQueue.add({ msg: 'Hello' });
```

```js Server B
const Queue = require('bull');

const sendQueue = new Queue('Server A');
const receiveQueue = new Queue('Server B');

receiveQueue.process(function (job, done) {
  console.log('Received message', job.data.msg);
  done();
});

sendQueue.add({ msg: 'World' });
```

</CodeGroup>

## How It Works

<Steps>
  <Step title="Server A sends a message">
    Server A adds a job to the "Server B" queue with the message "Hello"
  </Step>
  <Step title="Server B receives the message">
    Server B processes jobs from its receive queue and logs the message
  </Step>
  <Step title="Server B sends a response">
    Server B can send messages back to Server A using the same pattern
  </Step>
</Steps>

  <Card title="Asynchronous Communication" icon="clock">

  <Card title="Reliable Delivery" icon="shield-check">

  <Card title="Fault Tolerance" icon="rotate">
